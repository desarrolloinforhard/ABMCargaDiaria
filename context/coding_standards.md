# Coding Standards

- Codigo Python claro y modular.
- Evitar archivos gigantes.
- Separar responsabilidades por capa.
- Usar nombres en espanol para dominio de negocio si corresponde.
- Usar nombres tecnicos claros para infraestructura.
- Documentar decisiones importantes.
- No hardcodear rutas ni credenciales.
- Toda conexion debe ir luego por configuracion externa.
- Mantener UI basada en TkInforHard y no mezclar reglas de negocio en pantallas.
- Agregar tests enfocados cuando se introduzcan modelos, services o repositorios.
